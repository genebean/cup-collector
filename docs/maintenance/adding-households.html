Adding a Household

A household is a named collection shared by one or more users. Each household is backed by a PocketID group pair and a PocketBase record. Adding a second (or third) household involves five steps: create the groups, assign users, authorize the groups in the OIDC client, declare the household in Nix, and rebuild.

On this page
  1. Step 1 — Create the PocketID groups
  2. Step 2 — Add users to the groups
  3. Step 3 — Authorize the groups in the OIDC client
  4. Step 4 — Declare the household in the NixOS module
  5. Step 5 — Rebuild and verify

Step 1 — Create the PocketID groups

Every household needs two PocketID groups — one for owners (can mark cups as owned, upload photos) and one for viewers (read-only access). The group names must follow the exact naming convention the app expects:

cup_collector_{slug}_owner
cup_collector_{slug}_viewer

Where {slug} is a short identifier for the household — lowercase letters, numbers, and underscores only, no spaces.

  1. Open the PocketID admin UI → Groups → New Group.
  2. Set the Name to cup_collector_{slug}_owner (e.g. cup_collector_smith_family_owner).
  3. Set a human-readable Friendly Name (e.g. Cup Collector Smith Family Owner).
  4. Save, then repeat for cup_collector_{slug}_viewer.
Slug must match exactly The app strips the cup_collector_ prefix and _owner / _viewer suffix from the user's group claims at sign-in to recover the slug. A typo here means the household will never resolve. Double-check the name before saving.

Step 2 — Add users to the groups

  1. In PocketID admin → Users, open each user you want to add.
  2. Assign them to the appropriate group: _owner for full access, _viewer for read-only.
  3. A user can belong to groups for multiple households — they'll see a household switcher in Settings.

Step 3 — Authorize the groups in the OIDC client

PocketID's OIDC client has an Allowed User Groups list. If any groups are selected, only users in those groups can sign in to that client. You must add the new groups here or they won't appear in the groups claim of the OIDC token.

  1. Open PocketID admin → Applications → Cup Collector → Allowed User Groups.
  2. Check both cup_collector_{slug}_owner and cup_collector_{slug}_viewer.
  3. Click Save.
Existing users must sign out and back in The groups claim is embedded in the session token at sign-in time. Users who were already signed in won't see the new household until they sign out and authenticate again.

Step 4 — Declare the household in the NixOS module

Add an entry to the households list in your host's NixOS configuration. The slug must match exactly what you used in the PocketID group names.

services.cupCollector = {
  households = [
    {
      name = "Smith Family";
      slug = "smith_family";
    }
    {
      name = "Doe Family";   # new
      slug = "doe_family";   # must match cup_collector_doe_family_{role}
    }
  ];
};

The name is the display name shown in the app header and household switcher. The slug is the internal identifier used to match PocketID group claims to PocketBase records.

Step 5 — Rebuild and verify

  1. Apply the NixOS configuration: 
    sudo nixos-rebuild switch --flake .#your-hostname
  2. The cup-collector-pb-init oneshot service runs automatically after the rebuild and creates the new household record in PocketBase. It is idempotent — existing records are not modified.
  3. Confirm the record was created: open the PocketBase admin UI → Collections → households. The new entry should appear with the correct name and group_slug.
  4. Have a user in the new household sign in (or sign out and back in if already signed in). They should land in the app and see the correct household name in the header.
  5. Users belonging to multiple households will see a Switch household option in Settings.
Something wrong? If a user gets redirected to the access-denied page, check in order: (1) are they in a PocketID group with the correct name? (2) is that group checked in the OIDC client's Allowed User Groups? (3) did they sign out and back in after the group assignment? (4) does the slug in the Nix config exactly match the slug in the PocketID group name?
