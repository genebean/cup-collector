Backups

There are two sources of data that need to be kept safe: the PocketBase data directory and the curator's spreadsheet.

On this page
  1. What to back up
  2. Backup method
  3. Suggested backup frequency
  4. Restoring

What to back up

The PocketBase data directory is your most important backup target Everything — collection status, household record, and uploaded photos — lives here. If this is lost without a backup, you lose all ownership records and any photos you've uploaded. The catalog can be re-imported from the spreadsheet, but ownership history cannot be recovered.

PocketBase data directory: /var/lib/cup-collector/pb_data/

This contains:

  • The SQLite database (pb_data/data.db) — all collection records and household configuration
  • Uploaded files (pb_data/storage/) — catalog images and owner-uploaded photos

The curator's spreadsheet is the source of truth for the catalog. It lives outside this system (Google Sheets or local Excel). Keep it backed up separately — it can be used to re-import the catalog if the database is ever lost.

Backup method

If you're already backing up your homelab with Restic, Borgbackup, or a similar tool, add the PocketBase data directory to your backup paths:

# Example Restic backup (adjust paths and repo for your setup)
restic backup /var/lib/cup-collector/pb_data/

For a simple manual backup:

sudo tar -czf cup-collector-backup-$(date +%Y%m%d).tar.gz \
  /var/lib/cup-collector/pb_data/
PocketBase has a built-in backup feature In the PocketBase admin UI (via SSH tunnel at http://localhost:8090/_/) → Settings → Backups, you can trigger a backup and download it as a zip file. This is a good option for occasional manual snapshots before major changes.

Suggested backup frequency

  • Daily — if you mark cups owned regularly
  • Weekly — for less active periods
  • After any catalog import — take a manual backup immediately after a successful import

Restoring

  1. Stop the PocketBase container:
    sudo systemctl stop podman-cup-collector-pb
  2. Replace the data directory with your backup: 
    sudo tar -xzf cup-collector-backup-YYYYMMDD.tar.gz -C /
  3. Start the container again:
    sudo systemctl start podman-cup-collector-pb
  4. Verify by opening an SSH tunnel (ssh -L 8090:localhost:8090 yourserver) and checking the admin UI at http://localhost:8090/_/ — your collections and records should be present.

The Next.js app itself does not need to be backed up — it is rebuilt from source by Nix at any time. Only the PocketBase data directory is stateful.
