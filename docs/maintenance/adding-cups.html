Adding Cups to the Catalog

The cup catalog is maintained in a spreadsheet and imported via a script. Here's the full workflow from updating the spreadsheet to having new cups appear in the app.

The two roles in this workflow

  • Curator (the collaborator) — maintains the spreadsheet, adds new cups, exports CSV
  • Importer (the owner) — receives the CSV, runs the import script

The same person can do both. The split exists because the spreadsheet lives in Google Sheets or Excel, and the import script runs in a terminal.

Step 1 (Curator) — Update the spreadsheet

The spreadsheet has one row per cup, one column per field. The column order matters — it must match the CSV format exactly:

ColumnExampleNotes
citySan FranciscoRequired
regionCaliforniaState or province — leave blank if not applicable
countryUnited StatesRequired. Full name
country_codeUSRequired. 2-letter ISO code
seriesYou Are HereRequired. Must be consistent — check existing entries
year2021Required. 4-digit year
lat37.7749City centroid latitude — find via Google Maps
lng-122.4194City centroid longitude
image_urlhttps://example.com/cup.jpgPublic image URL — downloaded and stored during import
notesLimited edition 2021Optional freeform notes

Step 2 (Curator) — Export as CSV

  1. In Google Sheets: File → Download → Comma Separated Values (.csv)
  2. In Excel: File → Save As → CSV (Comma delimited)
  3. Send or share the file with the importer.

Step 3 (Importer) — Run the import script

Enter the dev shell first.
All commands must run inside nix develop. Never run npx or ts-node on the bare host. 
# 1. Enter the dev shell
nix develop

# 2. Preview what will change (no data is written)
npx ts-node scripts/import-cups.ts --file /path/to/cups.csv --dry-run

# 3. Review the output — check for unexpected creates or updates

# 4. Apply the import
npx ts-node scripts/import-cups.ts --file /path/to/cups.csv

The script is safe to re-run at any time. It uses upsert logic (match on city + series + year) — existing records are updated, new ones are created, nothing is duplicated.

Step 4 — Verify

  1. Open the PocketBase admin UI → Collections → cups — new entries should appear.
  2. Open the app — new cups should appear in Browse and on the map (if they have lat/lng).
