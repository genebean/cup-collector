Updating the App

Because Cup Collector is deployed as a Nix flake, updating it means bumping the flake input and rebuilding. There's no manual download or installation step.

The normal update flow

# In the genebean/dots repo:
# 1. Bump the cup-collector flake input to latest
nix flake update cup-collector

# 2. Check what changed (optional but recommended)
git diff flake.lock

# 3. Rebuild and switch
sudo nixos-rebuild switch --flake .#your-hostname

Checking what changed

Before rebuilding, you can see what commits are included in the update:

nix flake update cup-collector --commit-lock-file  # Updates and commits lock file
# Or: browse https://github.com/genebean/cup-collector/commits/main

Rolling back

If something breaks after an update, NixOS keeps the previous generation. Roll back instantly:

sudo nixos-rebuild --rollback switch

Or choose a specific generation:

sudo nixos-rebuild switch --flake .#your-hostname --rollback

This restores the previous version of the app, its config, and all NixOS services — no data is affected, only the running software.

When to update

  • Update promptly when the release notes mention a security fix or a bug that affects you.
  • Update at leisure for new features — the app is designed for long-term stability.
  • Cup Collector has no external runtime dependencies that go stale (other than the Google Places API, which is controlled by Google's versioning, not app releases).
No PocketBase data is affected by app updates.
The PocketBase data lives at /var/lib/cup-collector/pb_data/. An app update only changes the Next.js server binary and static assets. Your collection, photos, and household record are untouched.
