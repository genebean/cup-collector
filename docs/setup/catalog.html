Catalog Setup

The app needs a cup catalog before anyone can use it — the map, browse view, and owned/needed tracking are all driven by catalog records. This page covers the one-time initial import. For ongoing additions see Adding Cups.

Run everything inside nix develop Both build-catalog and import-cups are dev shell commands. Enter the shell first with nix develop, or call them non-interactively with nix develop -c <command>.

Step 1 — Generate the catalog CSV

build-catalog scrapes starbucks-mugs.com to produce a CSV containing all known cups across every series — You Are Here, Been There, Discovery Series, Relief, Icon Mini, and special editions. It fetches each cup's page to populate image URLs and release years, then writes cups.csv in the project root.

build-catalog

This takes a few minutes because it fetches each cup's page individually (concurrency=5 to be polite). You'll see a progress counter as it runs. When it finishes it prints a per-series summary.

Pages are cached locally in .scrape-cache/ after the first fetch. This keeps repeated runs fast and avoids hammering starbucks-mugs.com with redundant requests — the site is maintained by a single enthusiast, so being a considerate client matters. Delete the cache directory when you want a genuinely fresh scrape (e.g. after the site adds new cups).

To generate only one series — useful for testing or partial refreshes:

build-catalog --series "You Are Here"

Valid series names: You Are Here, Been There, Discovery Series, Relief, Icon Mini.

hobbydb_url is left blank The generated CSV has a hobbydb_url column that the scraper cannot populate automatically. If you have HobbyDB links, fill them in manually before importing. The import script preserves any hobbydb_url values already in the database — empty CSV cells won't overwrite hand-curated entries.

Skipped entries warning

At the end of its run, build-catalog may print a message like:

Skipped 4 entries with no coordinates (add to COORDS table to include):
  Disney Epcot 1 (You Are Here)
  Disney Epcot 2 (You Are Here)
  Disney Epcot 3 (You Are Here)
  Disney Parc Walt Disney Studios (You Are Here)

These are cups the scraper found on starbucks-mugs.com but could not place on the map because their city names don't match any entry in the coordinate lookup table. They are excluded from the CSV entirely — they won't appear on the map or in Browse until coordinates are added.

To include them, add their coordinates to app/src/lib/catalog-data.ts in the COORDS table. The key must be "CityName,CountryName" exactly as shown in the warning, and the value is [latitude, longitude]:

// In app/src/lib/catalog-data.ts, inside the COORDS block:
"Disney Epcot 1,France":                [48.8674, 2.7822],
"Disney Epcot 2,France":                [48.8674, 2.7822],
"Disney Epcot 3,France":                [48.8674, 2.7822],
"Disney Parc Walt Disney Studios,France":[48.8674, 2.7822],

After editing the file, re-run build-catalog and then import-cups to pick up the new entries. It's fine to skip this for now — cups without coordinates are rare edge cases and you can add them later.

Missing catalog images

build-catalog also reports cups whose page on starbucks-mugs.com has no product image:

  image_url resolved:   1967 / 1968
    [no image] Christmas Barcelona (You Are Here)

These cups will show a placeholder in the app. See Overriding scraped data below for how to supply a URL manually.

Overriding scraped data

Some cups have missing or incorrect data on starbucks-mugs.com that the scraper cannot fix automatically — a missing image, a wrong year, a note you want to add. Rather than patching the CSV by hand after every scrape, add a row to cups-overrides.csv in the project root.

The file has the same columns as cups.csv. Only name and series are required (they identify the cup); fill in only the fields you want to change and leave the rest blank:

# cups-overrides.csv
name,scope,venue_series,item_type,region,country,country_code,series,year,lat,lng,image_url,...
Christmas Barcelona,,,,,,,You Are Here,,,,https://example.com/xmas-barcelona.jpg,,,,,

cups-overrides.csv is tracked in git and applied automatically at the end of every build-catalog run. Blank cells are ignored — they never clear a scraped value.

Step 2 — Connect to PocketBase

import-cups reads POCKETBASE_URL, POCKETBASE_ADMIN_EMAIL, and POCKETBASE_ADMIN_PASSWORD from app/.env.local by default. Pass --prod to load app/.env.prod instead.

Create app/.env.prod with your production PocketBase connection details (this file is gitignored):

POCKETBASE_URL=http://your-server-lan-ip:8090
POCKETBASE_ADMIN_EMAIL=admin@example.com
POCKETBASE_ADMIN_PASSWORD=your-pocketbase-admin-password
PocketBase only listens on localhost by default. To reach it from your laptop over LAN without a tunnel, set pbBindIp to your server's LAN IP in the NixOS module and rebuild: 
pbBindIp = "192.168.1.10"; # your server's LAN IP
Then use that IP in app/.env.prod. If you haven't set pbBindIp (or prefer not to), open an SSH tunnel instead and omit --prod: 
ssh -L 8090:localhost:8090 yourserver

Step 3 — Preview the import

Dry-run mode shows exactly what will be created, updated, or skipped without writing anything:

import-cups --prod --dry-run

Review the output. If something looks wrong (unexpected errors, missing records), fix the CSV or check that PocketBase is reachable before proceeding.

Step 4 — Import

When the dry-run output looks right, apply it:

import-cups --prod

The script upserts records matched on name + series + year + item_type, so it's safe to re-run — it will not duplicate records, and a mug and an ornament with the same name, series, and year are kept as separate entries.

After upserting records, the script runs two automatic backfill passes:

  • Geo region backfill — any cup with coordinates but no region gets its state or province filled in automatically using geographic boundary data. This means you rarely need to set region manually for city-scope cups.
  • Sibling region backfill — if a cup still has no region after the geo pass, it inherits one from a same-city cup in the same series (useful for ornaments that share a city with a mug).

At the end of every import run, the script also lists any cups with no catalog image and notes whether an uploaded personal photo already covers them:

── Missing Catalog Images ──
  Christmas Barcelona (You Are Here) — no fallback, consider finding an image

If you find an image URL for a listed cup, add it to cups-overrides.csv (see Overriding scraped data above), re-run build-catalog, then re-import.

Step 5 — Verify

Open an SSH tunnel and check the PocketBase admin UI: Collections → cups should show your records. Once you sign in to the app, cups with coordinates will appear on the map and all cups will appear in Browse.
