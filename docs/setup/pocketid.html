PocketID Setup

Create the role groups, register Cup Collector, and add each household member. Groups must exist first because the app's access control is tied to them — you'll need to select them during app registration.

Step 1 — Create role groups

Cup Collector uses PocketID groups to determine what each user can do. Create these before registering the app:

  1. In the PocketID admin panel, go to User Groups → Create Group.
  2. Create a group named cup-owner. This role can mark cups owned and run imports.
  3. Create a group named cup-collaborator. This role can mark cups owned but cannot run imports.
  4. Create a group named cup-viewer. This role can browse and search but cannot make changes.
Group names are configurable
If you prefer different group names, set ROLE_GROUP_OWNER, ROLE_GROUP_COLLABORATOR, and ROLE_GROUP_VIEWER in your .env file. The defaults are the names above.

Step 2 — Register the application

  1. Navigate to Applications → Create Application.
  2. Fill in the fields as follows:

Logo / IconUpload public/icons/icon-192.png (192×192 PNG)
  1. Save the application. PocketID will generate a Client ID and Client Secret.
  2. Copy both values — you'll need them for the .env secrets file.

Step 3 — Configure application settings

After saving, open the application's settings and set the following toggles:

SettingValueWhy
Public clientOffThe app has a server-side component that keeps the client secret secure
PKCEOnRequired by Auth.js for additional authorization code security
Requires re-authenticationOffAllows persistent sessions without prompting for passkey on every visit
AccessAllow members of specific groups → select the three groups you just createdUsers not in any of these groups cannot sign in to the app at all

Step 4 — Add users and assign groups

Each person who will use Cup Collector needs a PocketID account and must be in at least one role group.

  1. In the PocketID admin panel, go to Users → Create User.
  2. Enter their name and email. They will set up a passkey on first login.
  3. Save the user, then open their profile and add them to the appropriate group (e.g. cup-owner for the primary account, cup-viewer for family members who only browse).

A user with no Cup Collector group will see an "access denied" page when they try to sign in.

Step 5 — Verify the "My Apps" launch icon

PocketID shows registered applications in each user's "My Apps" panel with a clickable icon. To verify:

  1. Have a user log in to PocketID.
  2. They should see a "Cup Collector" tile in their apps panel with the icon you uploaded.
  3. Clicking the tile should open https://cups.yourdomain.com.

If the icon doesn't appear, check that the Launch URL is saved correctly and that the icon image uploaded without errors.

Dev instance (local development)

To develop without touching your production PocketID, the dev shell includes a pocketid-serve command that starts a containerized PocketID instance on port 1411.

nix develop
pocketid-serve   # starts PocketID at http://localhost:1411

On first run, open http://localhost:1411, create an admin account (e.g. admin@example.com), then follow Steps 1–4 above using http://localhost:1411 as your PocketID base. For the redirect URI use:

http://localhost:3000/api/auth/callback/pocketid

Set up a passkey for the admin account. If you want the dev PocketID to persist across machine restarts without repeating setup, use PocketID's data export feature to save your configuration and re-import it when setting up on a new machine.

Localhost is a WebAuthn secure context
Passkeys work on http://localhost even without TLS. This is by design — browsers treat localhost as a secure origin.

Update your app/.env.local with the dev PocketID values:

POCKETID_ISSUER_URL=http://localhost:1411
POCKETID_CLIENT_ID=<client id from dev PocketID>
POCKETID_CLIENT_SECRET=<client secret from dev PocketID>
