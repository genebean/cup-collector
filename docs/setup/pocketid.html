PocketID Setup

Create your household's role groups, register Cup Collector as an OIDC application, and add users. Do this before deploying the app — you'll need the Client ID and Client Secret for the secrets file.

Choose your household slug first Before creating any groups, decide on a short identifier for your household — for example smith_family or our_collection. This slug is used in PocketID group names and ties those groups to your household record in PocketBase. Lowercase letters, numbers, and underscores only.

Step 1 — Create role groups

Cup Collector uses PocketID groups to determine what each user can do. In PocketID, go to User Groups → Create Group. Type the Friendly Name first — PocketID auto-generates the group name from it using underscores as word separators. Using slug smith_family as an example, create these two groups:

Friendly name (type this)Auto-generated group name (verify this)Access level
Cup Collector — Smith Family — Owner cup_collector_smith_family_owner Full access: browse, mark cups owned, record condition, upload photos, import catalog
Cup Collector — Smith Family — Viewer cup_collector_smith_family_viewer Read-only: browse and search the collection

After typing the friendly name, check that the auto-generated group name matches the value in the table exactly — this is what appears in OIDC claims and must match what the app expects. The convention is cup_collector_ prefix, your slug, then _owner or _viewer. The app ignores any group that doesn't start with cup_collector_.

Step 2 — Register the application

  1. In the PocketID admin panel, navigate to OIDC Clients → Create OIDC Client.
  2. Fill in the form. The fields and toggles all appear on the same creation screen:
FieldValue
NameCup Collector
Callback URLshttps://cups.yourdomain.com/api/auth/callback/pocketid
Logout Callback URLshttps://cups.yourdomain.com
Client Launch URLhttps://cups.yourdomain.com
LogoUpload app/public/icons/icon-512.png from the repo (512×512 PNG) — PocketID also accepts icon-192.png if you prefer the smaller size
Public ClientOff — the app has a server-side component that stores the client secret securely
PKCEOn — required by Auth.js for authorization code security
Requires Re-AuthenticationOff — allows persistent sessions without prompting for a passkey on every visit
Client IDLeave blank — PocketID auto-generates one
  1. Save. PocketID displays the generated Client ID and Client Secret — copy both for the secrets file during deployment.

Step 3 — Restrict access to your groups

The OIDC client page has an Allowed User Groups section below the form. It shows a table of all your PocketID groups with checkboxes. Check both groups you created in Step 1. If no groups are selected, all PocketID users can access the app — selecting groups here restricts access to only those members. Users not in either group will be denied at the PocketID login screen before they ever reach the app.

Step 4 — Add users and assign groups

Each person who will use Cup Collector needs a PocketID account and must be in one of the two role groups.

  1. In the PocketID admin panel, go to Users → Create User.
  2. Enter their name and email. They will set up a passkey on first login.
  3. Save the user, then open their profile and add them to the appropriate group: cup_collector_{slug}_owner for anyone who should be able to mark cups as owned, or cup_collector_{slug}_viewer for household members who only browse.

A user who isn't in either group will see an "access denied" page when they try to sign in.

Step 5 — Verify the "My Apps" launch icon

PocketID shows registered applications in each user's "My Apps" panel. To verify:

  1. Have a user log in to PocketID.
  2. They should see a "Cup Collector" tile in their apps panel with the icon you uploaded.
  3. Clicking the tile should open https://cups.yourdomain.com.

If the icon doesn't appear, check that the Launch URL is saved correctly and that the icon uploaded without errors.

Dev instance (local development)

To develop without touching your production PocketID, the dev shell includes a pocketid-serve command that starts a containerized PocketID instance on port 1411.

nix develop
pocketid-serve   # starts PocketID at http://localhost:1411

On first run, open http://localhost:1411, create an admin account, then follow Steps 1–4 above using http://localhost:1411 as your PocketID base. For the redirect URI use:

http://localhost:3000/api/auth/callback/pocketid

Set up a passkey for the admin account. Update your app/.env.local with the dev PocketID values:

POCKETID_ISSUER_URL=http://localhost:1411
POCKETID_CLIENT_ID=<client id from dev PocketID>
POCKETID_CLIENT_SECRET=<client secret from dev PocketID>
Passkeys work on localhost Browsers treat http://localhost as a secure context, so WebAuthn passkeys work without TLS during local development.
