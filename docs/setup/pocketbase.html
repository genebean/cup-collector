PocketBase Setup

After deploying, PocketBase runs at https://pb.yourdomain.com. Access the admin UI at https://pb.yourdomain.com/_/ to set up collections and create the household record.

Migrations run automatically
The PocketBase migration in pocketbase/migrations/001_initial_schema.js creates the three collections on first start. If migrations applied cleanly, you can skip to Step 3. Check the PocketBase logs to confirm.

Step 1 — Access the admin UI

  1. Open https://pb.yourdomain.com/_/ in your browser.
  2. On first run, PocketBase prompts you to create an admin account. Use a strong password and store it in your password manager.
  3. You're now in the PocketBase admin panel.

Step 2 — Verify collections

Go to Collections in the left sidebar. You should see three collections: households, cups, and owned_cups. If they're missing (migration didn't run), create them manually with these fields:

Step 3 — Set access rules

Both UI and database rules are required. The app hides write controls from viewers, but PocketBase rules are the security backstop. Don't skip this step.

For each collection, go to the collection settings → API Rules tab and set:

cups — catalog is read-only for all authenticated users; admin-only write:

  • List/View rule: @request.auth.id != "" (any authenticated user)
  • Create/Update/Delete rule: leave empty (admin only)

owned_cups — members can write; viewers cannot:

  • List/View rule: The app queries with a household filter — leave open for authenticated users or craft a rule matching household membership
  • Create rule: Only members — requires custom rule comparing @request.auth.id against household member_sub_1/member_sub_2
  • Delete rule: same as create

households — read for members and viewers; admin-only write:

  • List/View rule: authenticated users who appear in the record
  • Create/Update/Delete: leave empty (admin only)

Step 4 — Create the household record

  1. In the admin panel, go to Collections → households → New record.
  2. Set name to something friendly, e.g. "Our Collection".
  3. Set member_sub_1 to the PocketID sub of the first owner (found in Step 3 of the PocketID setup).
  4. Set member_sub_2 to the second owner's sub, if applicable.
  5. Leave viewer_subs as [] for now. Add viewer subs later when needed.
  6. Save.

To add a viewer later: edit the household record and add their sub to the viewer_subs JSON array, e.g. ["sub-value-here"].
