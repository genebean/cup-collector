Deployment

Cup Collector deploys via NixOS module. Everything — the Next.js app, PocketBase container, and nginx vhosts — is declared in your NixOS configuration.

All commands must run inside nix develop.
Never install Node, npm, or PocketBase directly on the host. The dev shell provides all tooling at pinned versions.

Step 1 — Add the flake input

In your NixOS config repo (genebean/dots), add Cup Collector as a flake input:

# In flake.nix inputs block:
cup-collector.url = "github:genebean/cup-collector";

Step 2 — Import the module

In the relevant host's NixOS configuration:

{ inputs, config, ... }: {
  imports = [ inputs.cup-collector.nixosModules.default ];

  services.cupCollector = {
    enable        = true;
    domain        = "cups.yourdomain.com";
    migrationsDir = inputs.cup-collector.packages.${pkgs.system}.migrations;
    # envFile points to your secrets file — managed by sops-nix or similar
    envFile       = config.sops.secrets."cup-collector-env".path;
    # pbDomain = "pb.yourdomain.com";  # optional — omit to keep PocketBase internal-only
    # port    = 3000;                  # these are all defaults
    # pbPort  = 8090;
    # dataDir = "/var/lib/cup-collector";
  };
}

Step 3 — Create the secrets file

The envFile must contain all required environment variables. Create it using your secrets manager (sops-nix, agenix, etc.) with the contents from .env.example:

POCKETID_ISSUER_URL=https://id.yourdomain.com
POCKETID_CLIENT_ID=cup-collector-abc123
POCKETID_CLIENT_SECRET=your-client-secret
AUTH_SECRET=generate-with-gen-auth-secret-in-nix-develop
NEXTAUTH_URL=https://cups.yourdomain.com
POCKETBASE_URL=http://localhost:8090
POCKETBASE_ADMIN_EMAIL=admin@example.com
POCKETBASE_ADMIN_PASSWORD=your-pocketbase-admin-password
GOOGLE_PLACES_API_KEY=your-google-places-key

Generate AUTH_SECRET with the gen-auth-secret command inside nix develop:

nix develop
gen-auth-secret

Step 4 — Rebuild

sudo nixos-rebuild switch --flake .#your-hostname

This will:

  • Build the Next.js app as a Nix package
  • Pull the PocketBase OCI image
  • Start both services
  • Configure nginx vhosts with Let's Encrypt certificates

Step 5 — Get the npmDepsHash

The first build will fail if npmDepsHash in flake.nix is a placeholder. Nix will print the correct hash in the error output. Copy it:

# The error output will include a line like:
#   got:    sha256-abc123...
# Copy that value into flake.nix as npmDepsHash, then rebuild.

Verify services are running

systemctl status cup-collector
systemctl status podman-cup-collector-pb  # or docker-cup-collector-pb

Check logs if something isn't working:

journalctl -u cup-collector -f
journalctl -u podman-cup-collector-pb -f

Common first-deployment issues

  • ACME certificate not issued — check that port 80 is open and DNS is pointing to your server. Let's Encrypt needs to reach http://cups.yourdomain.com/.well-known/acme-challenge/.
  • PocketBase container won't start — check the data directory permissions: ls -la /var/lib/cup-collector/. The pb_data directory should exist and be writable by the container.
  • Auth redirect error — double-check the NEXTAUTH_URL matches exactly what's in the PocketID redirect URI setting, including the protocol.
  • npmDepsHash mismatch — run nix build, copy the hash from the error, update flake.nix, rebuild.
