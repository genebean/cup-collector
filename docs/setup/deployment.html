Deployment

Cup Collector deploys via NixOS module. Everything — the Next.js app, PocketBase container, and Nginx vhosts — is declared in your NixOS configuration.

All commands must run inside nix develop Never install Node, npm, or PocketBase directly on the host. The dev shell provides all tooling at pinned versions.

Step 1 — Add the flake input

In your NixOS config repository, add Cup Collector as a flake input:

# In flake.nix inputs block:
cup-collector = {
  url = "github:genebean/cup-collector";
  inputs.nixpkgs.follows = "nixpkgs";
};

Step 2 — Import the module

In the relevant host's NixOS configuration:

{ inputs, config, pkgs, ... }: {
  imports = [ inputs.cup-collector.nixosModules.default ];

  services.cupCollector = {
    enable            = true;
    domain            = "cups.yourdomain.com";
    pocketidIssuerUrl = "https://id.yourdomain.com";
    migrationsDir     = inputs.cup-collector.packages.${pkgs.stdenv.hostPlatform.system}.migrations;
    households        = [
      {
        name = "Our Collection";
        slug = "our_collection"; # Must match the slug part of the PocketID group names
      }
    ];
    # envFile points to your secrets file — managed by sops-nix or similar
    envFile           = config.sops.secrets.cup_collector_env.path;
    # pbDomain        = "pb.yourdomain.com"; # optional — omit to keep PocketBase off the public internet
    # useDnsValidation = true;               # default: true — set false to use HTTP-01 challenge instead
    # maxBodySize      = "0";               # default: 0 (no limit) — set e.g. "10m" to cap uploads
    # pbBindIp        = "127.0.0.1";        # default: localhost-only; set to a LAN IP for direct access without SSH tunnel
    # port    = 3000;                        # these are the defaults
    # pbPort  = 8090;
    # dataDir = "/var/lib/cup-collector";
  };

  sops = {
    defaultSopsFile = ./secrets.yaml;
    secrets.cup_collector_env.restartUnits = [
      "cup-collector-pb-init.service"
      "cup-collector.service"
    ];
  };
}

Step 3 — Create the secrets file

Generate a random AUTH_SECRET first — you'll paste it into the secrets file below. From inside nix develop:

nix develop
gen-auth-secret

Or without the dev shell:

openssl rand -base64 32

Create the secrets file using your secrets manager (sops-nix, agenix, etc.). The module generates NEXTAUTH_URL, POCKETBASE_URL, POCKETID_ISSUER_URL, and NEXT_CACHE_DIR automatically, and household records are declared as a module option — only these six secrets are needed:

POCKETID_CLIENT_ID=cup-collector-abc123
POCKETID_CLIENT_SECRET=your-client-secret
AUTH_SECRET=<output of gen-auth-secret>
POCKETBASE_ADMIN_EMAIL=admin@example.com
POCKETBASE_ADMIN_PASSWORD=your-pocketbase-admin-password
GOOGLE_PLACES_API_KEY=your-google-places-key
Choose your PocketBase admin credentials here POCKETBASE_ADMIN_EMAIL and POCKETBASE_ADMIN_PASSWORD are credentials you'll create — pick them now and write them into the secrets file. After deployment, the init service creates the superuser automatically. Save them in your password manager.

Step 4 — Rebuild

sudo nixos-rebuild switch --flake .#your-hostname

This will:

  • Build the Next.js app as a Nix package
  • Pull the PocketBase OCI image
  • Start both services
  • Configure Nginx vhosts with Let's Encrypt certificates

Verify services are running

systemctl status cup-collector
systemctl status podman-cup-collector-pb  # or docker-cup-collector-pb

Check logs if something isn't working:

journalctl -u cup-collector -f
journalctl -u podman-cup-collector-pb -f
PocketBase is now running Once services are healthy, PocketBase is up and migrations have applied. Continue to the next step to create the superuser and set up your household.

Common first-deployment issues

  • ACME certificate not issued yet — DNS validation runs in the background after first deploy and can take a minute or two. Check progress with: 
    journalctl -t acme-order-renew-cups.yourdomain.com-start
    Substitute your actual domain. If it's still running, wait for it to complete before concluding something is wrong.
  • PocketBase container won't start — check data directory permissions: ls -la /var/lib/cup-collector/. The pb_data directory should exist and be writable by the container.
  • Auth redirect error — verify that NEXTAUTH_URL matches the Redirect URI in PocketID exactly, including the protocol (https://).
  • Migrations didn't run — check journalctl -u podman-cup-collector-pb on startup. Confirm migrationsDir is set correctly in the module config.
