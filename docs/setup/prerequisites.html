Prerequisites

Everything you need in place before deploying Cup Collector. Most of this infrastructure you likely already have if you're running a NixOS homelab.

Infrastructure

NixOS server Required
A machine running NixOS, reachable from the internet. The app and PocketBase both run here. Your existing homelab host works fine — the Cup Collector module adds to whatever is already running.
Nginx Required
Nginx must be running on the host, configured via the NixOS services.nginx module. The Cup Collector NixOS module adds its own virtual hosts automatically.
ACME / Let's Encrypt Required
TLS is required for PWA installation and geolocation. NixOS handles this automatically via security.acme. You need a valid domain pointed at your server and port 80 open for the ACME HTTP challenge.
PocketID (self-hosted OIDC) Required
PocketID must already be running and accessible at a public URL (e.g. https://id.yourdomain.com). Cup Collector uses it as the identity provider. If you don't have it yet, see the PocketID documentation to set it up first.
OCI container runtime Required
PocketBase runs as a container. The project is developed and tested with Podman — enable virtualisation.podman.enable = true in your NixOS configuration. Docker works too (virtualisation.docker.enable = true), but Podman is what the dev shell and docs assume.

Domain names

You need at least one subdomain pointing to your server's public IP:

  • cups.yourdomain.com — the Next.js app (required)
  • pb.yourdomain.com — PocketBase admin UI (optional — omit to keep PocketBase off the public internet)

You can use any subdomain names; update the module options to match.

Keep PocketBase internal Omitting pbDomain from the module config keeps PocketBase off the public internet. You access the admin UI via SSH tunnel when needed. This is the recommended production setup.

External services

Google Places API key Required
Used server-side to find nearby Starbucks stores. Enable the Places API (New) in Google Cloud Console, then create a key under APIs & Services → Credentials. Personal use will almost certainly stay in the free tier. The key is kept server-side only — it is never exposed to the browser.
GitHub account Optional
The app can be hosted on any Git forge. GitHub is assumed in the docs deployment workflow. If you use Forgejo or GitLab, adjust the Actions workflow accordingly.

Local development (optional)

To run the app locally for development or testing, you only need:

  • Nix installed on your machine (nix develop provides everything else — Node, PocketBase, all tooling)
  • Podman available on your host (virtualisation.podman.enable = true on NixOS) — used by pocketid-serve to run the dev PocketID container
  • A app/.env.local file with real values — PocketID client credentials, PocketBase admin email/password, and Google Places API key

The dev shell provides pb-serve, pocketid-serve, and dev-next to run the full stack locally without touching any production services.
