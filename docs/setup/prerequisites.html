Prerequisites

Everything you need in place before deploying Cup Collector. Most of this infrastructure you likely already have if you're running a NixOS homelab.

Infrastructure

NixOS server Required
A machine running NixOS, reachable from the internet. The app and PocketBase both run here. Your existing homelab host is fine — the module adds to whatever is already running.
nginx Required
nginx must be running on the host, configured via the NixOS services.nginx module. The Cup Collector NixOS module adds its own virtual hosts automatically.
ACME / Let's Encrypt Required
TLS is required for PWA installation and geolocation. NixOS handles this automatically via security.acme. You need a valid domain pointed at your server and port 80 open for the ACME challenge.
PocketID (self-hosted OIDC) Required
PocketID must already be running and accessible at a public URL (e.g. https://id.yourdomain.com). Cup Collector uses it as the identity provider. If you don't have it yet, see the PocketID documentation to set it up first.
OCI container runtime Required
PocketBase runs as a container. Enable virtualisation.docker.enable = true or virtualisation.podman.enable = true in your NixOS config.

Domain names

You need two subdomains, both pointing to your server's public IP:

  • cups.yourdomain.com — the Next.js app
  • pb.yourdomain.com — PocketBase

You can use any subdomain names; just update the module options accordingly.

External services

Google Places API key Required
Used server-side to find nearby Starbucks stores. Enable the Places API (New) in Google Cloud Console, then create a key under APIs & Services → Credentials. Personal use will almost certainly stay in the free tier. The key is kept server-side only — never exposed to the browser.
GitHub account (for the repo and docs) Optional
The app can be hosted on any Git forge. GitHub is assumed in the docs deployment workflow. If you use Forgejo or GitLab, adjust the Actions workflow accordingly.

Local development (optional)

To run the app locally for development or testing, you only need:

  • Nix installed on your machine (nix develop provides everything else)
  • Podman available on your host (virtualisation.podman.enable = true on NixOS) — used by pocketid-serve to run the dev PocketID container
  • A app/.env.local file with real values (PocketID client credentials, PocketBase admin email/password)

The dev shell provides pb-serve, pocketid-serve, and dev-next commands to run the full stack locally without touching any production services. You do not need to install Node, npm, or PocketBase separately.
