First Run Checklist

After deploying, work through this list in order. Each item includes a verification step so you know it's working before moving on.

  • Both services are running
    Run: systemctl status cup-collector and systemctl status podman-cup-collector-pb. Both should show "active (running)".
  • PocketBase is accessible
    Open https://pb.yourdomain.com/_/ in a browser. You should see the PocketBase admin login page.
  • The three collections exist in PocketBase
    Log in to the admin UI and check Collections. households, cups, and owned_cups should all be present with the correct fields. If not, follow the PocketBase setup guide.
  • Household record created
    In the admin UI → Collections → households → verify one record exists with a name. If not, create it: just set a name and save.
  • Import the cup catalog
    Enter the dev shell and run the import script: 
    nix develop

# Preview what will change (no data written)
import-cups --file your-cups.csv --dry-run

# Review the output, then apply:
import-cups --file your-cups.csv
    Verify in PocketBase admin UI → Collections → cups that records appear.
  • Sign in on first phone
    Open https://cups.yourdomain.com in Safari, tap "Sign in with PocketID", and log in with your passkey. You should land on the map screen with cup pins visible. Check Settings to confirm your role shows correctly.
  • Sign in on second phone
    Repeat for the second household member. Both should be signed in with their own accounts, each showing their appropriate role.
  • Test real-time sync
    With both phones open, have one person mark a cup as owned. Within 1–2 seconds, the other phone's map should update — the orange pin should turn green without any manual refresh.
  • Install as PWA on each phone
    In Safari, tap Share → "Add to Home Screen" → Add. The Cup Collector icon should appear on the home screen. Tap it — the app should open full-screen with no Safari UI. See Getting Started for the full walkthrough.
  • Test location and nearby stores
    Open the app on your phone with a real GPS signal. Allow location access. The map should zoom to your position and show blue Starbucks pins nearby (if any are within 2km). Tap a blue pin to verify the "Get directions" link opens Apple Maps.

Local development

To run the full stack locally without touching production, the dev shell provides three commands:

nix develop

pb-serve          # PocketBase on http://localhost:8090
pocketid-serve    # PocketID on http://localhost:1411
dev-next          # Next.js dev server on http://localhost:3000

Each runs in its own terminal. On first run, open http://localhost:8090/_/ and http://localhost:1411 to complete the initial setup for each service (create admin accounts, register the OIDC app, create role groups). See the PocketID setup guide and PocketBase setup guide for details.

Dev data is separate from production
PocketBase stores data in pocketbase/pb_data/ and PocketID stores data in pocketid/data/. Both directories are gitignored. The dev PocketID generates its own encryption key in pocketid/encryption_key (also gitignored, mode 600).
