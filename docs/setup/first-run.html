First Run Checklist

Verify the infrastructure is healthy after completing all setup steps — PocketID, Deployment, and Catalog Setup. Once everything checks out, sign in yourself and work through Getting Started — you'll catch any issues that way, and you'll already know the app when household members ask questions.

Infrastructure

  • Both services are running
    Run: systemctl status cup-collector and systemctl status podman-cup-collector-pb. Both should show "active (running)".
  • Init service ran — superuser and household records created
    The cup-collector-pb-init service creates the PocketBase superuser and all household records automatically. Verify it succeeded: 
    systemctl status cup-collector-pb-init
    It should show "active (exited)". If it failed, check journalctl -u cup-collector-pb-init for details.
  • All three collections exist in PocketBase
    Open an SSH tunnel (ssh -L 8090:localhost:8090 yourserver), log in to http://localhost:8090/_/, and check Collections. households, cups, and owned_cups should all be present. If missing, check the migrations log: journalctl -u podman-cup-collector-pb.
  • PocketID groups have users assigned
    Confirm that groups cup_collector_{slug}_owner and/or cup_collector_{slug}_viewer exist in PocketID and have users assigned. See the PocketID setup guide.
